# CUFA 부록 B — Claude 활용 투자 분석 가이드

AI를 투자 분석에 활용하는 실전 방법을 다룬다. Claude(claude.ai)와 Claude Code를 중심으로.

# 1. Claude를 투자 분석에 쓰는 원칙

AI가 잘하는 것: 데이터 수집·정리, 사업보고서 요약, 산업 개요 초안, 재무 테이블 생성, 문장 교정, 구조/일관성 체크.

AI에 맡기면 안 되는 것: 투자 논거 설정, 핵심 가정 결정(매출 성장률·GPM), 피어 선정·멀티플 판단, 리스크 평가, 최종 투자의견.

원칙: AI는 "보조 분석가"지 "의사결정자"가 아니다. 분석의 속도를 10배 올려주지만, 판단의 질은 분석가에게 달려 있다.

# 2. Claude.ai 실전 프롬프트

## 2-1. 산업 분석 프롬프트

다음 프롬프트를 claude.ai에 입력하면 산업 개요 초안이 나온다.

프롬프트 예시:

"한국 화장품 ODM 산업을 분석해줘. 다음 항목을 포함해:

1) 글로벌/한국 시장 규모와 CAGR

2) 핵심 성장 드라이버 3가지

3) 밸류체인 구조 (원료→ODM→브랜드→유통)

4) 경쟁구도 (주요 기업 M/S, 진입장벽)

5) 한국 기업의 글로벌 경쟁력

6) 투자 시사점

수치는 가능한 최신 데이터로, 출처를 명시해줘."

이 결과를 2주차 산업 분석의 초안으로 활용한다. 반드시 DART·IR 자료로 교차 검증해야 한다.

## 2-2. 사업보고서 분석 프롬프트

DART에서 사업보고서 PDF를 다운받아 Claude에 업로드하고:

"이 사업보고서의 'II. 사업의 내용'을 읽고 다음을 정리해줘:

1) 사업부별 매출 구성과 비중

2) 각 사업부의 산업 현황 요약

3) 경쟁사 언급 (직접/간접)

4) 설비투자(CAPEX) 규모와 방향

5) R&D 투자 비중

6) 경영진이 강조하는 핵심 경쟁력

7) 투자 관점에서 주의할 리스크"

## 2-3. 재무 데이터 정리 프롬프트

"다음 재무 데이터를 엑셀에 넣을 수 있는 테이블로 정리해줘:

[여기에 네이버 금융/CompanyGuide에서 복사한 데이터 붙여넣기]

포함할 항목: 매출, 영업이익, 순이익, 총자산, 총부채, 자본, 영업CF, CAPEX, FCF.

YoY 성장률도 계산해줘.

GPM, OPM, NPM, ROE, 부채비율, 이자보상배율도 계산해줘."

## 2-4. 실적 추정 프롬프트

"다음 가정을 기반으로 A사의 2026~2027년 실적을 추정해줘:

매출 성장률: Base 15%, Bull 25%, Bear 5%

GPM: 29% → 31%로 개선 가정

판관비 증가율: 12%

CAPEX/매출: 5%

유효세율: 22%

유통주식수: 2,000만주 (자사주 소각 반영)

IS 요약 테이블 + FCF 계산 + Bull/Base/Bear 시나리오별 EPS를 구해줘."

## 2-5. 밸류에이션 프롬프트

"A사의 밸류에이션을 수행해줘.

1) Peer Multiple: 피어 3곳(한국콜마, 코스메카코리아, Intercos)의 PER/EV-EBITDA 제시.

2) DCF: WACC 10%, 영구성장률 2% 가정. FCF는 [여기에 수치].

3) PBR: 2027E BPS와 ROE 기반.

4) Football Field 테이블로 종합.

5) 리스크-리워드 비율 계산."

## 2-6. 보고서 초안 프롬프트

"2~6주차 과제를 바탕으로 기업분석 보고서 초안을 작성해줘.

구조: 표지 → Executive Summary → 산업분석 → 비즈니스분석 → 재무분석 → 실적추정 → 밸류에이션 → 리스크 → 결론.

각 섹션의 마지막에 '투자 시사점'을 한 줄 넣어줘.

분량: 15~20페이지."

## 2-7. 보고서 검증 프롬프트

"이 보고서의 논리 일관성을 체크해줘:

1) 산업 분석의 성장률과 실적 추정의 매출 성장률이 일관되는가?

2) 경쟁우위 주장이 재무 데이터(GPM, OPM)로 뒷받침되는가?

3) 리스크 분석이 Bear Case와 연결되는가?

4) 숫자 불일치가 있는가? (IS의 EPS와 밸류에이션의 EPS가 같은가?)

5) 중복되거나 불필요한 서술이 있는가?"

# 3. Claude Code 활용

## 3-1. 재무 데이터 자동 수집

Claude Code에서 Python 스크립트로 재무 데이터를 자동 수집한다.

OpenDartReader 설치: pip install opendartreader

기본 사용:

```python

import OpenDartReader as odr

dart = odr.OpenDartReader('YOUR_API_KEY')

# 삼성전자 재무제표

fs = dart.finstate('005930', 2025)  # 2025년 연결재무제표

# 특정 항목 추출

revenue = fs[fs['account_nm']=='매출액']['thstrm_amount'].values[0]

```

이걸 확장하면 여러 기업의 재무 데이터를 한번에 수집하여 엑셀/CSV로 내보낼 수 있다. 부록 C(DART API 가이드)에서 상세히 다룬다.

## 3-2. 피어 비교 자동화

```python

peers = ['005930', '000660', '042700']  # 삼성전자, SK하이닉스, 한미반도체

for code in peers:

fs = dart.finstate(code, 2025)

# OPM, ROE 계산 후 테이블에 추가

```

결과를 엑셀로 내보내면 4주차 과제(피어 비교 비율 테이블)가 자동으로 완성된다.

## 3-3. 고빈도 데이터 자동 수집

관세청 수출 데이터, DRAM 가격 등을 주기적으로 수집하는 스크립트를 Claude Code로 만들 수 있다. 5주차에서 다룬 "주간 데이터 모니터링 루틴"의 자동화.

# 4. 주의사항

(1) AI가 생성한 숫자는 반드시 원본(DART, 네이버 금융, CompanyGuide)으로 교차 검증한다. AI는 확신 있게 틀릴 수 있다.

(2) AI가 생성한 산업 분석·보고서 초안은 "출발점"이다. 본인의 판단으로 수정·보완해야 한다. AI 출력을 그대로 제출하면 차별화된 분석이 아니다.

(3) 프롬프트가 구체적일수록 결과가 좋다. "분석해줘"보다 "다음 6가지 항목을 포함하여 분석해줘"가 10배 나은 결과를 낸다.

(4) 기밀 정보(미공개 IR 자료, 내부 데이터)를 AI에 입력하지 않는다. CUFA 교안 범위에서는 공개 데이터만 사용하므로 이 이슈는 크지 않지만, 향후 실무에서는 주의해야 한다.

# 5. 핵심 프롬프트 5종 (주차별 활용)

10가지 시나리오를 5가지 핵심 프롬프트로 압축했다. [대괄호] 안만 바꿔서 사용한다.

## 프롬프트 1: 산업 분석 + 기업 선정 (W1~W2)

"[산업명] 산업을 분석해줘. 다음 항목을 순서대로 포함:

1) 글로벌/한국 시장 규모 (2025년 기준, 원화/달러)

2) 2020~2025년 CAGR, 2025~2030년 전망 CAGR

3) 핵심 성장 드라이버 3가지 (각 3줄)

4) 밸류체인 구조 (원자재→부품→완제품→유통 순서로)

5) 밸류체인 각 단계의 주요 기업 (한국 기업 중심)

6) 경쟁구도: 상위 5사 M/S, 진입장벽 수준

7) 산업생명주기 (도입/성장/조정/성숙/쇠퇴 중 어디)

8) 핵심 리스크 2가지

9) 투자 시사점 3줄

수치는 가능한 최신 데이터로, 출처를 명시해줘."

변형: 기업 선정이 목적이면 "이 산업에서 시총 3,000억~3조, OPM 8%+ 기업을 5개 추천해줘"를 끝에 추가. 사업보고서(PDF)를 업로드하면 "이 사업보고서의 'II. 사업의 내용'을 읽고 사업부별 매출 구성, 경쟁사, CAPEX, CAPA, P×Q 정보를 정리해줘"로 변형.

## 프롬프트 2: 재무 데이터 수집·비교 (W3~W4)

"다음 재무 데이터를 엑셀에 바로 붙여넣을 수 있는 테이블로 정리해줘:

[여기에 네이버 금융/CompanyGuide에서 복사한 데이터]

포함 항목 (5개년): 매출, 매출원가, 매출총이익, 판관비, 영업이익, 이자비용, 세전이익, 법인세, 순이익, 영업CF, CAPEX, FCF, 총자산, 총부채, 자본총계, 매출채권, 재고, 매입채무.

추가 계산: YoY 성장률, GPM, OPM, NPM, ROE, 부채비율, 이자보상배율, CCC.

피어 비교(W4 변형): [관심기업], [피어A], [피어B]의 3개년 비율을 비교 테이블로 만들고, 듀폰 분해(ROE = NPM × 회전율 × 레버리지) + '관심기업이 Peer 대비 어디가 강하고 약한지' 핵심 시사점 5줄."

## 프롬프트 3: 실적 추정 (W5)

"다음 가정을 기반으로 [기업명]의 2026~2027년 실적을 추정해줘:

가정: [매출 성장률 Base/Bull/Bear, GPM, 판관비 증가율, CAPEX/매출, 감가상각비, 유효세율, 유통주식수를 나열]

산출:

1) IS 요약 (매출→영업이익→순이익→EPS)

2) CF 요약 (영업CF→FCF)

3) Bull/Base/Bear 시나리오별 EPS

4) 기대 EPS (확률 가중)

5) 민감도 테이블 (매출성장률 x GPM)

테이블 형태로 정리해줘."

## 프롬프트 4: 밸류에이션 (W6)

"[기업명]의 밸류에이션을 수행해줘.

1) Peer Multiple: 피어 [A], [B], [C]의 시총/순이익/PER → Peer 평균 PER 적용 → 목표주가.

2) DCF: WACC [X]%, 영구성장률 [Y]%. FCF: [수치 나열]. TV → 기업가치 → 목표주가. 민감도 테이블.

3) Football Field: PER/EV-EBITDA/DCF를 Bear/Base/Bull 범위로 종합. 리스크-리워드.

4) 투자의견 + 근거 3줄 + 핵심 리스크 3개."

## 프롬프트 5: 보고서 통합·검증·Q&A (W7~W8)

보고서 초안: "2~6주차 과제를 바탕으로 [기업명] 보고서 초안을 작성해줘. 구조: 표지→ES→산업→비즈니스→재무→추정→밸류에이션→리스크→결론→Appendix. 각 섹션 끝에 투자 시사점 1줄. 15~20페이지."

논리 검증(변형): "이 보고서의 논리 일관성을 검증해줘. 산업↔추정 일관성, 경쟁우위↔재무 뒷받침, 리스크↔Bear Case 연결, 숫자 불일치, 투자논거 한 줄 요약 가능 여부. Pass/Fail로."

Q&A 대비(변형): "[기업명] 투자 피칭에서 나올 수 있는 질문 20개 + 모범 답변(3줄). 산업 5개/재무 5개/밸류 5개/약점 공격 5개. 가장 어려운 3개에 나쁜 답변 예시도."

# 6. Claude Code 상세 가이드

## 6-1. Claude Code란

터미널에서 Claude와 대화하면서 코드를 실행하는 도구다. "코스맥스 5개년 재무 데이터를 DART에서 수집해서 엑셀로 내보내줘"라고 말하면 자동으로 Python 스크립트를 작성하고 실행한다.

설치: npm install -g @anthropic-ai/claude-code

실행: 터미널에서 claude 입력.

## 6-2. 재무 데이터 자동 수집

Claude Code에서 다음과 같이 요청한다:

"OpenDartReader로 코스맥스(192820)의 2021~2025년 연결 재무제표를 수집해줘. 매출, 영업이익, 순이익, 영업CF, CAPEX를 추출하고 엑셀로 저장해줘."

Claude Code가 자동으로:

1. opendartreader 패키지 확인/설치

2. API Key 설정

3. 5년치 재무제표 API 호출

4. 핵심 항목 추출

5. pandas DataFrame 생성

6. 엑셀 파일로 저장

이 과정이 수동으로 하면 30분, Claude Code로 하면 2분.

## 6-3. 피어 비교 자동화

"코스맥스(192820), 한국콜마(161890), 코스메카코리아(241710)의 2023~2025년 OPM/ROE/부채비율을 비교 테이블로 만들어줘. 조건부 서식까지 적용한 엑셀 파일로 저장."

## 6-4. 고빈도 데이터 수집

"관세청 수출 데이터에서 화장품(HS코드 3304) 월별 수출 금액을 2024년 1월~2026년 2월까지 수집해줘. 추이 차트도 그려줘."

## 6-5. 공시 모니터링 자동화

"삼성전자(005930)의 2026년 1월 이후 공시 중 '자사주' 또는 '배당' 관련 공시만 필터링해서 리스트로 보여줘."

"다음 기업에 대해 보고서용 데이터 패키지를 만들어줘:

피어: 한국콜마(161890), 코스메카코리아(241710)

# 7. AI 활용의 한계와 윤리

## 7-1. AI가 틀리는 영역

재무 수치: AI는 특정 기업의 정확한 재무 수치를 "기억"에서 답할 때 틀릴 수 있다. 반드시 DART/CompanyGuide 원본으로 교차 검증한다.

최신 데이터: AI의 학습 데이터에는 시차가 있다. 최근 분기 실적, 최신 공시는 AI가 모를 수 있다. 웹 검색 기능을 활용하거나 직접 확인한다.

업종 특수성: AI가 일반론으로 답할 때 업종 특수성을 놓칠 수 있다. 예를 들어 "부채비율이 높으면 위험하다"는 일반론이지만, 은행은 부채비율이 높은 게 정상이다.

미래 예측: AI는 미래를 예측할 수 없다. "이 기업이 오를까요?"라는 질문에 AI가 답할 수 없다. AI는 분석의 도구이지 예측 기계가 아니다.

## 7-2. AI 생성 콘텐츠의 윤리

AI가 작성한 초안을 그대로 제출하는 것은 학습 효과가 없다. AI는 "속도"를 올려주는 도구이지, "판단"을 대체하는 것이 아니다.

CUFA 보고서에서 AI 활용 원칙:

(1) 데이터 수집·정리: AI 활용 적극 권장. 시간 절약 효과가 크다.

(2) 초안 작성: AI 초안을 출발점으로 사용하되, 본인의 판단으로 수정·보완해야 한다.

(3) 투자 논거·가정·의견: 반드시 본인이 직접 작성. AI 출력을 그대로 사용하면 차별화된 분석이 아니다.

(4) 논리 검증: AI에 검증을 요청하는 것은 좋지만, 최종 판단은 본인이 한다.

## 7-3. 프롬프트 작성의 핵심 원칙

(1) 구체적으로. "분석해줘"보다 "다음 6가지 항목을 포함하여 분석해줘"가 10배 좋은 결과를 낸다.

(2) 구조를 지정한다. "테이블 형태로", "각 항목 3줄로", "번호를 매겨서" 같은 출력 형식 지정.

(3) 맥락을 준다. "CUFA 3기 신입교육 과제를 위한 것이다. 대상은 비전공 대학생이다." 같은 배경 정보.

(4) 단계별로 요청한다. 한번에 모든 것을 요청하기보다, 산업분석 → 재무정리 → 추정 → 밸류에이션 순서로 단계별 대화가 더 좋은 결과를 낸다.

(5) 교차 검증을 요청한다. "이 분석에서 틀릴 수 있는 부분을 스스로 점검해줘."

CUFA 3기 부록 B. Claude 활용 투자 분석 가이드.

AI는 분석의 속도를 10배 올려주지만, 판단의 질은 분석가 본인에게 달려 있다.

# 8. 실전 워크플로우 — 주차별 Claude 사용 타임라인

## 8-1. 1주차 (30분)

1. 관심 산업 탐색 (10분): "한국 시장에서 2026년 주목할 산업 5개를 추천해줘" → 결과를 참고하여 관심 산업 1개 선택.

2. 관심 기업 후보 (10분): "선택 산업 내 시총 3000억~3조 기업 중 OPM 8%+ 기업 5개" → CompanyGuide에서 교차 확인.

3. 최근 뉴스 브리핑 (10분): "선택 기업의 최근 3개월 주요 뉴스와 이슈를 정리해줘" → 투자 포인트 초안.

## 8-2. 2주차 (45분)

1. 산업 분석 초안 (20분): 5-2 프롬프트 사용 → 결과를 DART IR 자료로 교차 검증.

2. 사업보고서 분석 (15분): PDF 업로드 + 5-3 프롬프트 → P×Q 분해 정보 추출.

3. 밸류체인 정리 (10분): "이 기업의 밸류체인에서 전방/후방 기업을 5곳씩 정리해줘" → CUFA 밸류체인 맵과 대조.

## 8-3. 3~4주차 (30분)

1. 재무 데이터 정리 (10분): 5-4 프롬프트로 5개년 데이터 테이블 생성 → 엑셀에 복사.

2. 비율 자동 계산 (10분): 5-5 프롬프트로 피어 비교 테이블 + 듀폰 분해.

3. 이익의 질 체크 (10분): "이 기업의 이익의 질을 5단계로 체크해줘" → 적색 신호 확인.

## 8-4. 5~6주차 (45분)

1. 실적 추정 (20분): 5-6 프롬프트로 IS/CF 추정 + 3시나리오.

2. 밸류에이션 (15분): 5-7 프롬프트로 Peer Multiple + DCF + Football Field.

3. 민감도 체크 (10분): "WACC과 성장률을 바꿔가면서 목표주가가 어떻게 변하는지 테이블로 보여줘".

## 8-5. 7~8주차 (60분)

1. 보고서 초안 (30분): 5-8 프롬프트로 전체 초안 생성.

2. 논리 검증 (15분): 5-9 프롬프트로 일관성 체크.

3. Q&A 대비 (15분): 5-10 프롬프트로 예상 질문 20개 + 모범 답변.

총 소요: 8주간 약 3.5시간. AI 없이 하면 20시간+. AI가 데이터 수집·정리를 담당하고, 분석가(학생)는 판단과 해석에 집중하는 구조.

## 8-6. 주의: AI 의존 vs AI 활용의 차이

AI 의존 (나쁜 예):

"코스맥스를 분석해서 보고서 써줘" → AI가 쓴 보고서를 그대로 제출.

AI 활용 (좋은 예):

"코스맥스의 산업 현황을 정리해줘" → 결과를 DART로 교차 검증 → 본인 판단으로 투자 시사점 도출 → "이 시사점이 논리적인지 체크해줘" → 최종 본인 작성.

차이: 전자는 "AI가 분석한 것"이고, 후자는 "내가 AI 도움을 받아 분석한 것"이다. 면접에서 "이 보고서에서 본인이 가장 고민한 부분은?"이라는 질문에 답할 수 있어야 한다.
