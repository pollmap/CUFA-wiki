## 0. 문체 3원칙

### ① 강의형 서사 (Narrative Lecture)
- AI 티 나는 구조적 요약 금지. **불릿 리스트 남용 금지.**
- "그러면 왜 그런가?" → 답변 → 근거 → 결론 **구어체 질문·응답** 구조 기본.
- 독자를 **"여러분"**으로 호명. "살펴보도록 하겠습니다", "주목해야 합니다", "생각해 보죠".
- 장황한 분사·관형절 대신 **짧은 문장 연결**. "그래서 ~. 그러나 ~. 따라서 ~."
- **좋은 예** (유튜브 종교학 강의 톤):
  > "본시 메타노이아라는 용어는 신약 시대 이전, 그러니까 고대 헬라스 문명권에서는 그 용례를 찾기가 쉽지 않은 것이었으며 구약에서도 회개만을 가리키는 특별한 단어가 있었다 보긴 쉽지 않습니다."
- **나쁜 예** (AI 요약):
  > "메타노이아는 신약 용어이며 구약에서는 흔치 않다."

### ② 원어·원문 인용 (Primary Source)
- 핵심 개념에 **반드시** 원어 병기: `DCF(Discounted Cash Flow, 현금흐름할인법)`, `WACC(가중평균자본비용, Weighted Average Cost of Capital)`.
- 한자권 전문용어는 한자 병기: `가치투자(價値投資)`, `대수의 법칙(大數의 法則)`.
- 공식 인용은 **논문·경전·법 조항 번호까지**: 자본시장법 제9조 제21항, IFRS 9.B5.4.1, Fama & French (1992) *J. Finance*, BIS *Basel III: Finalising post-crisis reforms* (2017).

### ③ 비교·대조·비판 (Comparative Critical)
- 단일 개념 설명 금지. 반드시 **비교 대상** 포함.
  - DCF → 상대가치(PER/EV-EBITDA)와 비교
  - MPT → Risk Parity / Black-Litterman / HRP 와 비교
  - 한국 자본시장법 → 미국 SEC / EU MiFID 와 비교
- 비판적 시각 포함: 한계·엣지케이스·반례. "하지만 이런 방식은 ~ 경우엔 무너진다."

## 2. 난이도 뱃지 (백준 모델)

프론트매터 `difficulty` 값으로 **5단계** 분류. 백준의 Bronze/Silver/Gold/Platinum/Diamond 차용.

| 뱃지 | 값 | 대상 | 예시 |
|---|---|---|---|
| 🥉 Bronze | `beginner` | 비전공자·완전 입문 | 재무상태표란? / 주식이란? |
| 🥈 Silver | `basic` | 전공 기초 | 수익성 지표 / ROE 해석 |
| 🥇 Gold | `intermediate` | 전공 중급 | DCF 기본 / WACC 계산 |
| 💎 Platinum | `advanced` | 심화·실무 | Black-Litterman 유도 / 헤징 실전 |
| 🏅 Diamond | `research` | 논문급·리서치 | Stochastic DCF / Bayesian Factor Models |

**선수지식(prerequisites)** 프론트매터 필수. PrerequisiteGraph 자동 렌더.

## 3. 금지 표현 (AI 티 검출 대상)

Evaluator CI가 `DRAFT: false` 문서에서 감지 시 **실패** 처리.

- "다음과 같은 ~이 있습니다" (→ 서사로 풀어쓰기)
- "~에 대해 알아보겠습니다" **자체는 OK, 남발 금지**
- 불릿 5개 이상 연속 (→ 문단으로 전환)
- "매우", "굉장히", "아주" 반복 3회+
- 같은 단락 내 같은 조사/어미 3회+ 반복
- 인용 없는 "연구에 따르면", "전문가들은" → 반드시 출처
- "첫째, 둘째, 셋째" 습관적 나열 (→ "먼저 ~. 그런데 ~. 따라서 ~.")
- 영어 약어 풀이 없이 사용 (WACC/CAPM/CAGR 첫 등장 시 원어 필수)

- 학점·자격증·영어·인턴의 "별점 ★★★★★" 기반 서술도 출처·근거 명시

## 6. 예시 — 좋은 문단 vs 나쁜 문단

**❌ 나쁨 (AI 요약형)**
```
DCF는 기업가치 평가 방법입니다. 다음과 같은 특징이 있습니다.
- 미래현금흐름을 할인합니다
- WACC을 사용합니다
- 장기가치를 반영합니다
장점: 이론적으로 가장 엄밀한 방법입니다.
단점: 가정에 민감합니다.
```

**✅ 좋음 (강의형 서사)**
```
DCF(Discounted Cash Flow, 현금흐름할인법)은 기업이 앞으로 만들어낼 현금흐름을
지금 가치로 환산해 합산하는 밸류에이션 기법입니다. 그런데 미래의 돈은 지금의
돈과 같은 가치를 갖지 않습니다. 왜일까요? 1년 뒤 1만 원과 지금의 1만 원 중
어느 쪽이 더 값진가를 생각해 보면 됩니다. 지금 1만 원으로 우리는 예금을
들거나 투자를 할 수 있지만, 1년 뒤의 1만 원은 그동안 불어났을 기회를 잃은
돈이거든요. 그래서 Damodaran(2012)은 *Investment Valuation* 3판에서 이런

여기서 $r_e$는 자기자본비용, $r_d$는 세전 타인자본비용입니다. 문제는 이
공식이 *상수* WACC를 가정한다는 점입니다. 자본구조가 기간마다 바뀌는 기업에
```

## 7. 해설자 톤 (Narrator Tone) — 객관성 ≠ 중립성

v2에서 강의형 서사(§0-①)를 한 층 더 구체화한 층이다. **"단정하지 않고 구조를 보여주는 큐레이터"** 포지션을 유지한다. 리서치·분석 섹션에서 특히 엄격히 적용한다.

### 7.1 근본 원칙

1. **출발점의 구분 명시**: 공식 자료 기반인가, 분석 영역인가를 문서 도입부에서 드러낸다.
   - ✅ "이 섹션은 DART 공시를 1차 자료로 한다."
   - ✅ "이 파트는 분석적 영역으로 — 복수 출처의 교차 검증에 기반한다."
2. **객관성 ≠ 중립성**: 객관적(검증 가능) ≠ 중립적(양쪽을 동등 분량). 둘을 분리해 명시한다.
3. **정답 유보**: 해석이 여러 갈래인 사안에는 청자에게 판단을 넘긴다. "결국 이 자산을 들고 갈지 정리할지는 개인의 리스크 감당 범위에 따라 결정될 수밖에 없다고 생각합니다."
4. **메타 노출**: 왜 이런 방식으로 설명하는지를 드러낸다. "이 문서의 뼈대가 되는 자료는 ~이다."

### 7.2 담화 구조 (3-layer 샌드위치)

```
[Layer 1] 방법론 — "이 섹션에서는 ~ 방식으로 접근한다"
[Layer 2] 자료·수치 — "[출처 A]의 수치는 ~이고, [출처 B]는 ~이다"
[Layer 3] 해석 여러 갈래 — "한쪽은 ~, 다른 한 쪽은 ~. 결국 ~은 미결."
```

하나의 주장을 못 박지 않는다. 자료 → 해석 A → 해석 B → 미결정 → 청자 판단 이양 순서.

### 7.3 대립 구조 의무

§0-③의 "비교·대조" 확장. **분석·전망 단락에는 반드시 "반면 / 반대로 / 한쪽은~ 다른 한 쪽은~" 중 하나**를 배치한다. 대립이 없는 단일 해석은 **"분석 단락"이 아니라 "정의 단락"**으로 재분류한다.

### 7.4 종결 어미 분포 (분석·해석 단락에 한정)

| 어미 | 기능 | 권장 비중 |
|---|---|---|
| `~입니다 / ~습니다` | 기본 전달 | 35% |
| `~이죠 / ~거죠` | 친근 확인·요약 | 20% |
| `~로 보입니다 / ~인 것으로 보입니다` | 판단 완화 | 15% |
| `~라고 생각합니다` | 의견 명시 | 10% |
| `~일 가능성이 높습니다` | 확률적 추론 | 5% |
| `~일 수도 있습니다 / ~라고 말할 수도 있습니다` | 가능성 제시 | 5% |
| `~이겠죠 / ~소리겠죠` | 추론 여운 | 5% |
| `~잖아요 / ~거든요` | 공유 전제·배경 | 5% |

**금지 종결**: `~이다` / `~답은 ~이다` / `~는 분명하다`. 마지막 문장조차 `~라고 생각합니다`로 닫는다. 관찰 가능한 사실에 한해 `~는 명확합니다`는 허용.

### 7.5 헤지 4계층

> 헤지 밀도 기준: **3~4문장마다 1개**. 수치·자료 인용은 헤지 없이 직설, 해석·판단에만 헤지.

| 계층 | 목적 | 예시 |
|---|---|---|
| **인식적** | 주장 확신도 낮추기 | `~로 보입니다`, `~일 가능성이 높습니다`, `~로 해석될 수 있습니다` |
| **출처 기반** | 책임을 자료로 이동 | `보고서는 ~라고 설명합니다`, `~라고 분석하고 있습니다`, `~라고 서술합니다` |
| **범위** | 적용 범위 제한 | `단기적으로는 ~`, `이 시점 기준으로 ~`, `처음 주제의 틀을 잡을 때만 ~` |
| **메타** | 자기 발화 대상화 | `저는 정답이 아니라는 거죠`, `제가 하는 소리가 맞다라는 게 아닙니다`, `저도 한쪽에 치우친 얘기를 할 수도 있습니다` |

### 7.6 자료 인용 3단계 (수치 제시 규율)

```
1. 원수치     → "2025년 KOSS해운 1일 처리량은 약 N 배럴"
2. 분모 명시  → "해상 수입 총량 1일 기준 약 1,580만 배럴 대비"
3. 비율 환산  → "원유 전체 수입의 약 5.4%에 해당한다"
```

**계산 범주 이동 가능성** 명시: "해상 수입만 따질 경우", "정제 원유로 한정 시", "Forward / Trailing / TTM 기준".

### 7.7 자문자답 + 수사의문문

리듬 장치. 해석 단락에서 1~2회 권장.

```
[현상 제시]  "베네수엘라 공급이 막혔는데 러시아산 원유가 오히려 내려갔다."
[수사의문]  "그럼 왜 내려갔을까요?"
[자료 인용] "보고서는 이렇게 설명합니다 — ..."
[재진술]    "그러니까 ~라는 거죠."
[귀결]      "결국 ~로 이어졌습니다."
```

### 7.8 수치 양가 해석 관용구

| 관용구 | 사용 맥락 |
|---|---|
| "많다면 많고 적다면 적은 애매한 수치죠" | 중간값 수치 |
| "타격을 줬을 수도 있지만 크게 줬냐 하긴 애매한 수치죠" | 영향 평가 |
| "단편적으로 정답을 말하기 어려운 상황입니다" | 미확정 사안 |
| "시간이 좀 더 지나야 객관적 평가가 가능하다고 생각합니다" | 장기 미결 |

### 7.9 Anti-patterns (검증 실패 유도)

§3 금지 표현의 연장. Evaluator가 감지 시 **FAIL**.

| 패턴 | 이유 |
|---|---|
| `~이다`, `~는 분명하다` | 단정 → 정답 유보 원칙 충돌 |
| `엄청난`, `충격적인`, `경악스러운`, `미친` | 감정 과장 → 관찰자 톤 파괴 |
| `반드시`, `무조건`, `절대` (확신 강화어) | 단정 강화 → 헤지 시스템과 충돌 |
| `~는 잘못됐다`, `~는 정당하다`, `~는 비열하다` | 도덕 단죄 → "구조 기술, 가치 평가 금지" 위반 |
| `ㅋㅋ`, `!`, `~~`, 이모티콘 | 관찰자 톤 파괴 |
| `제 결론은 ~입니다. 왜냐하면 ~` | 결론 강요 → 판단 청자 이양 위반 |

### 7.10 자기 한계·반례 수용 구문

논지를 세운 뒤 **스스로 반례를 도입**한다. 단순 겸양이 아니라 논증 강도 조절 장치.

- "물론 ~할 수 있지만"
- "다만 ~"
- "물론 제 생각이 틀릴 수도 있습니다"
- "~라고 치더라도"
- "물론 ~가 무조건 약한 건 아닙니다"

### 7.11 청자 호칭 규율

- ✅ `여러분`, `보시는 분들`, `궁금하신 분들`, `~하시는 분들`
- ❌ `당신`, `시청자분`, `구독자분`, `유저` (거리감 무너짐 / 판매자 톤)

### 7.12 클로징 패턴

| 유형 | 표현 |
|---|---|
| 판단 유보 | "결국 보시는 분들이 내릴 수밖에 없다고 생각합니다." |
| 한계 인정 | "저는 정답이 아니라는 거죠." |
| 시간 유보 | "시간이 좀 더 지나야 객관적 평가가 가능하다고 생각합니다." |
| 관점 확장 선언 | "여러분들의 관점을 넓히려는 거지 답을 제시하려는 건 아닙니다." |

## 8. 카피 체크리스트 (분석·리서치 단락 전용)

문서 저장 전 다음 10항 확인. Evaluator 수동 보조용.

- [ ] 문서 도입부에 **출발점 구분** (공식 자료 / 분석 영역) 명시
- [ ] 수치·자료를 **최소 1개 이상** 출처와 함께 제시
- [ ] **"반면 / 반대로 / 한쪽은~"** 대립 구조 ≥ 1회
- [ ] 주장 문장 중 **`~로 보입니다 / ~이겠죠 / ~일 가능성이 높습니다`** 헤지 등장
- [ ] **"한마디로 / 그러니까 / 결국"** 요약 진입어로 핵심 재정리
- [ ] **메타 헤지** 1회 이상: "제가 하는 소리가 맞다라는 게 아닙니다" / "저는 ~라고 보고 있습니다"
- [ ] 종결 어미가 `~입니다` 일변도가 아니라 `~이죠 / ~거죠`가 혼재
- [ ] **확신 강화어** (엄청/완전/진짜/!/ㅋㅋ/반드시/무조건) 전부 제거
- [ ] **도덕 단죄**·**확정 결론** 없음
- [ ] 청자에게 판단을 넘기는 **클로징** 존재

## 10. 철학 요약 — 한 문장

> **"수사 장식 없이 방법론·출처·대립 구조를 먼저 보여주고, 결론은 청자에게 넘긴다."**

이 한 줄이 §0(3원칙)과 §7(해설자 톤) 전체를 관통한다. 표면 어휘만 흉내 내서는 질감이 나오지 않는다. **원칙을 먼저 체화한 다음 어휘가 따라오는 순서**로 작성한다.
