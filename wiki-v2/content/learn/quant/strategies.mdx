# 6종 전략 프리셋

## 전략 계층 구조

Luxon Quant System은 6종의 전략 프리셋을 3개 Tier로 구분한다. Tier가 낮을수록 학술적 근거가 풍부하고 실전 검증이 완료된 전략이다.

| Tier | 전략 | 상태 | Capital Ladder |
|------|------|------|---------------|
| 1 | 한국 멀티팩터 | 실전 운용 | SEED 이상 |
| 1 | 크립토 캐리 (펀딩레이트) | 실전 운용 | SEED 이상 |
| 2 | 페어트레이딩 | 백테스트 완료 | PAPER |
| 2 | 글로벌 매크로 | 백테스트 완료 | PAPER |
| 3 | 볼 타이밍 | 연구 단계 | 미배포 |
| 3 | ML 멀티알파 | 연구 단계 | 미배포 |

## Tier 1: 실전 운용 전략

### 1. 한국 멀티팩터 (Monthly Rebalancing)

한국 주식시장에서 학술적으로 검증된 팩터 프리미엄을 체계적으로 수확하는 전략이다.

#### 전략 개요

| 항목 | 내용 |
|------|------|
| 유니버스 | KOSPI200 + KOSDAQ150 |
| 리밸런싱 | 월간 (매월 첫 거래일) |
| 포지션 | 롱 온리 (공매도 불가) |
| 종목 수 | 15-30개 |
| 벤치마크 | KOSPI200 TR |

#### 4대 팩터

```python
FACTOR_WEIGHTS = {

    "value": {

    },
    "low_vol": {
        "weight": 0.20,
        "metric": "60D_realized_vol",

    },

```

#### 한국 시장 팩터 프리미엄 (실증 연구)

| 팩터 | 연간 프리미엄 | 출처 | 기간 |
|------|-------------|------|------|
| Value (PBR) | +4.2% | Kang et al. (2019, KAIST) | 2000-2018 |
| Momentum (12-1) | +3.8% | Kim & Lee (2017, KRX) | 2005-2016 |
| Quality (ROE) | +3.1% | Choi (2020, SNU) | 2003-2019 |
| Low Volatility | +2.7% | Park et al. (2018) | 2000-2017 |
| **합산 (단순)** | **~10-12%** | | 비용 차감 전 |

주의: 팩터 프리미엄은 과거 실적이며 미래를 보장하지 않는다. **비용 차감 후 실질 알파**가 핵심이다.

#### 코드 예시

```python
from kis_backtest.core.pipeline import QuantPipeline, PipelineConfig

pipeline = QuantPipeline(PipelineConfig(
    total_capital=5_000_000,
    target_vol=0.10,
    kelly_fraction=0.5,       # Half-Kelly 필수
    rebalance_frequency="monthly",
))

result = pipeline.run(
    factor_scores=factor_scores,
    optimal_weights=bl_weights,
    returns_dict=returns_dict,
)

print(f"Sharpe: {result.sharpe:.2f}")
print(f"CAGR: {result.cagr:.1%}")
print(f"Max DD: {result.max_dd:.1%}")
```

#### 연간 비용 시뮬레이션

```
월간 리밸런싱 = 12 RT/년
RT 비용 = 0.23% (KOSPI 대형주 기준)
연간 비용 = 12 x 0.23% = 2.76%

Gross Alpha: ~10%
After-Cost Alpha: ~7.2%
벤치마크 대비 초과수익: 양의 알파 기대
```

### 2. 크립토 캐리 (펀딩레이트 차익)

현물 매수 + 무기한 선물 숏으로 펀딩레이트를 수취하는 시장 중립 전략이다.

#### 전략 개요

| 항목 | 내용 |
|------|------|
| 거래소 | Upbit (현물) + Binance (선물) |
| 대상 | BTC, ETH |
| 포지션 | 시장 중립 (Delta = 0) |
| 수익 원천 | 펀딩레이트 (보통 양수) |
| 거래세 | 0% (암호화폐) |
| 리밸런싱 | 8시간마다 (펀딩 정산 주기) |

#### 펀딩레이트 메커니즘

```
선물 가격 > 현물 가격 (Contango)
  --> 롱 포지션이 숏 포지션에게 펀딩레이트 지급
  --> 현물 롱 + 선물 숏 = 펀딩레이트 수취 (Carry)

펀딩레이트 평균: 0.01% / 8시간
  --> 연환산: 0.01% x 3 x 365 = 10.95%
  --> 시장 중립이므로 방향 리스크 없음
```

#### 리스크 요인

| 리스크 | 설명 | 대응 |
|--------|------|------|
| 펀딩 음전환 | 하락장에서 펀딩레이트 마이너스 | 음의 펀딩 시 포지션 해제 |
| 거래소 리스크 | 거래소 해킹/파산 | 분산 예치, 최소 잔고 유지 |
| 베이시스 리스크 | 현물-선물 괴리 급변 | 스프레드 모니터링, 임계값 설정 |
| 마진 콜 | 급변동 시 선물 마진 부족 | 충분한 마진 비율 유지 (3x+) |

## Tier 2: 백테스트 완료 전략

### 3. 페어트레이딩

통계적 차익거래(Statistical Arbitrage) 전략이다. 공적분(Cointegration) 관계에 있는 두 종목의 스프레드가 평균으로 회귀하는 성질을 이용한다.

#### 한국 시장 제약과 대응

한국 시장에서 개인 투자자는 공매도가 불가능하다. 따라서 전통적인 롱-숏 페어트레이딩 대신 변형 전략을 사용한다.

| 방법 | 설명 | 장단점 |
|------|------|--------|
| 롱-롱 비중차 | 두 종목 모두 매수, 비중 조절 | 완전 중립 불가 |
| 인버스 ETF | 인버스 ETF로 숏 효과 | 일간 리밸런싱 비용 |
| ETF 활용 | 섹터 ETF 간 페어 | 유동성 양호 |

#### OU Process 기반 구현

```python
class PairTrader:
    """Ornstein-Uhlenbeck 기반 페어트레이딩"""

    def __init__(
        self,
        entry_z: float = 2.0,
        exit_z: float = 0.5,
        stop_z: float = 4.0,
    ):
        self.entry_z = entry_z
        self.exit_z = exit_z
        self.stop_z = stop_z

    def fit_ou(self, spread: pd.Series) -> dict:
        """OU 프로세스 파라미터 추정"""
        # dX = theta * (mu - X) * dt + sigma * dW
        spread_lag = spread.shift(1).dropna()
        spread_diff = spread.diff().dropna()

        # OLS 회귀: dX = a + b * X_{t-1}
        from scipy.stats import linregress
        slope, intercept, _, _, _ = linregress(
            spread_lag, spread_diff
        )

        theta = -slope         # 평균회귀 속도
        mu = intercept / theta  # 장기 평균
        sigma = spread_diff.std()

        half_life = np.log(2) / theta if theta > 0 else np.inf

        return {
            "theta": theta,
            "mu": mu,
            "sigma": sigma,
            "half_life_days": half_life,
        }

    def generate_signal(
        self, spread: float, mu: float, sigma: float
    ) -> str:
        """Z-Score 기반 시그널"""
        z = (spread - mu) / sigma
        if abs(z) > self.stop_z:
            return "STOP_LOSS"
        elif z > self.entry_z:
            return "SHORT_SPREAD"
        elif z < -self.entry_z:
            return "LONG_SPREAD"
        elif abs(z) < self.exit_z:
            return "CLOSE"
        return "HOLD"
```

MCP 도구 `stat_arb_ou_fit`을 활용하면 실시간 OU 파라미터를 추정할 수 있다.

## Tier 3: 연구 단계 전략

### 5. 볼 타이밍

HMM(Hidden Markov Model)으로 시장 레짐을 탐지하고, 고변동 레짐에서 주식 -> 채권/현금으로 이동하는 전략이다.

#### 2-State HMM

```python
class VolTimingStrategy:
    """HMM 레짐 기반 볼 타이밍"""

    def __init__(self, n_states: int = 2):
        self.n_states = n_states
        self.model = None

    def fit(self, returns: pd.Series) -> None:
        """HMM 학습"""
        from hmmlearn import hmm
        self.model = hmm.GaussianHMM(
            n_components=self.n_states,
            covariance_type="full",
            n_iter=100,
        )
        self.model.fit(returns.values.reshape(-1, 1))

    def predict_regime(self, returns: pd.Series) -> int:
        """현재 레짐 예측 (0: 저변동, 1: 고변동)"""
        states = self.model.predict(returns.values.reshape(-1, 1))
        return states[-1]

    def allocate(self, regime: int) -> dict:
        """레짐별 자산 배분"""
        if regime == 0:  # 저변동 (Bull)
            return {"equity": 0.80, "bond": 0.15, "cash": 0.05}
        else:            # 고변동 (Bear/Crisis)
            return {"equity": 0.20, "bond": 0.50, "cash": 0.30}
```

#### GARCH 변동성 예측 보조

```python
from arch import arch_model

# GARCH(1,1) 모델
model = arch_model(returns, vol="Garch", p=1, q=1)
result = model.fit(disp="off")

# 1일 앞 변동성 예측
forecast = result.forecast(horizon=1)
```

### 6. ML 멀티알파

Marcos Lopez de Prado의 Meta-Labeling 기법을 활용하여 여러 팩터의 시그널을 ML 모델로 결합하는 전략이다.

#### Meta-Labeling 컨셉

```
기존 방식: 팩터 --> 가중합 --> 시그널
ML 방식:   팩터 --> Primary Model (방향) --> 맞을 확률 예측
                  --> Secondary Model (사이즈) --> 확률에 비례한 포지션
```

#### 과최적화 방지

ML 전략에서 과최적화는 치명적이다. 반드시 다음 원칙을 따른다.

| 원칙 | 설명 |
|------|------|
| Purged K-Fold CV | 학습/검증 데이터 시간 오염 방지 |
| Embargo | 접경 구간 제거 (정보 누출 차단) |
| Feature Importance | SFI(Single Feature Importance) 검증 |
| Walk-Forward OOS | 최종 검증은 반드시 Walk-Forward |

```python
from sklearn.ensemble import RandomForestClassifier

# Meta-Labeling 예시
primary_signal = factor_composite > threshold  # 1차 시그널

# 2차 모델: 1차 시그널이 맞을 확률
meta_model = RandomForestClassifier(
    n_estimators=100,
    max_depth=5,           # 과적합 방지
    min_samples_leaf=50,   # 최소 샘플 수
    class_weight="balanced",
)
meta_model.fit(X_train, y_meta_label)

# 포지션 사이즈 = 1차 시그널 방향 x 2차 모델 확률
prob = meta_model.predict_proba(X_test)[:, 1]
position_size = primary_signal * prob * half_kelly_fraction
```

## 한국 시장 제약

한국 주식시장에서 퀀트 전략을 운용할 때 반드시 고려해야 할 구조적 제약이다.

### 주요 제약 사항

| 제약 | 상세 | 영향 | 대응 |
|------|------|------|------|
| 공매도 금지 (개인) | 기관/외국인만 가능 | 숏 전략 불가 | 인버스 ETF, 비중 0 클리핑 |
| 증권거래세 0.20% | 매도 시 부과 | 고빈도 전략 비용 과다 | 월간 이하 리밸런싱 |
| 양도세 (대주주) | 250만원 초과 22% | 세후 수익 감소 | 세금 최적화, 분산 투자 |
| 30% 가격 제한폭 | 상하한가 제한 | 테일 리스크 차단 | 손절 주문 미체결 가능 |
| T+2 결제 | 매도 후 2일 대기 | 자금 회전 제약 | 현금 비율 관리 |
| 장중 시간 제한 | 09:00-15:30 | 야간 리스크 헤지 불가 | 선물/해외 ETF 활용 |

### 거래비용 모델 상세

```python
# 한국 주식 거래비용 모델 (2025년 기준)
KOREA_COST_MODEL = {
    "commission_buy": 0.00015,   # 매수 수수료 0.015%
    "commission_sell": 0.00015,  # 매도 수수료 0.015%
    "securities_tax": 0.0020,    # 증권거래세 0.20%
    "slippage_large": 0.0005,    # 대형주 슬리피지 5bps
    "slippage_small": 0.0015,    # 소형주 슬리피지 15bps
}

def round_trip_cost(is_large_cap: bool = True) -> float:
    """Round-Trip 비용 계산"""
    m = KOREA_COST_MODEL
    slippage = m["slippage_large"] if is_large_cap else m["slippage_small"]
    return (
        m["commission_buy"] + m["commission_sell"]
        + m["securities_tax"]
        + slippage * 2
    )
    # 대형주: 0.23%, 소형주: 0.43%

def annual_cost(turnover: int, is_large_cap: bool = True) -> float:
    """연간 거래비용"""
    return turnover * round_trip_cost(is_large_cap)
    # 월간 리밸런싱(12x), 대형주: 2.76%
    # 월간 리밸런싱(12x), 소형주: 5.16%
```

### 비용이 전략에 미치는 영향

| 전략 | 연간 회전율 | 연간 비용 (대형주) | Gross Alpha 필요 |
|------|-----------|-------------------|-----------------|
| 멀티팩터 (월간) | 12x | 2.76% | > 5% |
| 페어트레이딩 (주간) | 52x | 11.96% | > 15% |
| 글로벌 매크로 (분기) | 4x | 0.92% | > 3% |
| 크립토 캐리 | N/A | 0% (거래세 없음) | > 0% |
